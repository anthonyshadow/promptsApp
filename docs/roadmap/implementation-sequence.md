# Implementation Sequence

## Purpose

Turn `docs/roadmap/next-steps-checklist.md` into the dependency-ordered execution tracker for future Codex implementation prompts.

## Status Rules

- `not_started`: no meaningful implementation exists yet.
- `in_progress`: partial, mocked, schema-only, or UI-only implementation exists.
- `complete`: acceptance criteria are met for the current MVP scope, with evidence.
- `blocked`: work needs an external decision, service, credential, or upstream dependency before a useful implementation prompt can run.

## Update Protocol

When a future implementation prompt completes work:

1. Update the row status and risk notes here.
2. Update `docs/roadmap/checklist-status.md` with evidence and validation.
3. If the work changes beta or launch readiness, update `private-beta-readiness.md`, `production-readiness.md`, or `launch-blockers.md`.
4. Preserve MVP scope: no enterprise SSO, SOC 2 workstreams, production routing, full CRM automation, or public CRM.
